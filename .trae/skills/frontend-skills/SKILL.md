# Frontend Skills

Use this skill when the task is primarily about designing, structuring, and implementing polished frontend experiences.

## When To Invoke

Invoke this skill when the user asks to:

- Build a frontend page, layout, or route
- Create a premium landing page or multi-page marketing site
- Create or update UI components and page systems
- Improve styling, responsiveness, motion, or accessibility
- Refactor frontend structure for maintainability
- Translate a brand or product brief into a complete frontend experience

## Working Style

1. Start from the user's business goals, audience, and brand position, not just a list of sections.
2. Inspect the existing frontend stack before making changes and reuse current patterns when the project already has them.
3. Prefer clean information architecture, reusable components, and consistent spacing and typography systems.
4. Keep accessibility, responsive behavior, and performance in scope by default.
5. Build interfaces that simplify complexity and guide users toward clear outcomes.
6. Avoid copying competitor layouts or wording even when a reference site is provided.

## Design Principles

- Think in systems, not isolated sections
- Favor premium restraint over visual noise
- Use strong hierarchy, whitespace, and clear reading paths
- Turn complex service information into understandable flows
- Balance authority, warmth, and precision in the copy and UI
- Make calls to action intentional and specific

## Premium Consulting Website Brief

When the user asks for a premium consulting website for cooperative housing societies, redevelopment, or real estate advisory, follow this brief unless the user overrides it.

### Positioning

- Brand type: productized consulting service
- Tone: confident, professional, intelligent, minimal, and persuasive
- Perception goals:
  - experts trusted with complex legal and operational work
  - technology-enabled consulting
  - premium, not mass-market

### Creative Direction

- Reimagine from first principles rather than mirroring any reference site
- Do not replicate layout, structure, wording, or visual style from competitor sites
- Aim for a premium SaaS-meets-consulting experience
- Visual inspiration can borrow qualities such as clarity, restraint, typography, and precision from modern product brands, but the result must remain original

### Design Language

- Clean layouts with a strong grid
- Large breathing space and structured composition
- Soft shadows and restrained glassmorphism where appropriate
- Subtle gradients, not loud or flashy color use
- Smooth micro-interactions and elegant motion
- Premium visual rhythm across hero, content bands, and CTAs

### Color And Type

- Primary direction: deep navy or blue
- Accent direction: gold or warm yellow
- Background direction: white with soft gradient overlays
- Neutrals: layered grays for cards, borders, and text hierarchy
- Typography direction: bold modern sans-serif for headings and clean readable body text

### UX Requirements

- Simplify dense or technical services into clear, digestible pathways
- Use cards, icons, timelines, diagrams, and structured content blocks to reduce cognitive load
- Create distinct journeys for developer-focused and society-member-focused audiences
- Reinforce trust using experience, project volume, domain expertise, and process clarity
- Make the consulting offer feel reliable, premium, and easy to understand

### Required Pages

1. Home
2. About
3. Services
4. For Developers
5. For Society Members
6. Redevelopment
7. Resources or Blog
8. Contact

### Home Page Expectations

- Hero with a sharp value proposition
- Trust indicators
- Services preview
- How-it-works visual flow
- Why choose us
- Stats counters
- Testimonials
- Strong CTA section
- Premium footer

### Other Page Expectations

- About: story, mission and vision, leadership cards, milestone timeline
- Services: grid-based service system with icons and clear explanations
- For Developers: problem-to-solution storytelling with tailored messaging
- For Society Members: simplified breakdown with clarity and reassurance
- Redevelopment: step-based timeline, benefits, and case-style narrative
- Resources or Blog: clean article card grid that supports thought leadership
- Contact: premium form, office details, and map placeholder

### Component System

Build reusable components such as:

- Navbar with a thoughtful mega menu or expanded navigation pattern
- Hero sections
- Service and feature cards
- Animated stats counters
- Timeline blocks
- Testimonials slider or marquee with restraint
- CTA sections
- Footer

### Motion And Interaction

- Smooth scrolling
- Fade-in on scroll
- Hover elevation
- Button micro-interactions
- Animated counters
- Subtle section transitions

Keep all motion elegant and restrained.

### Technical Defaults

- Preferred stack: HTML, Tailwind CSS, and vanilla JavaScript unless the project clearly benefits from minimal React
- Must be fully responsive
- Keep code clean, structured, and reusable
- Separate content, styling utilities, and interaction logic cleanly

### Output Expectations For This Brief

Provide or build:

- A sensible folder structure
- Complete page files
- Tailwind-based styling approach
- JavaScript for interactions and animations
- Reusable sections and components

## Delivery Checklist

- Identify the framework and styling approach in use
- Map the audience, conversion goals, and trust signals
- Define reusable sections, layout patterns, and visual hierarchy
- Build the requested UI with minimal clutter and strong consistency
- Preserve existing design patterns when working in an established project
- Verify the edited files for obvious build or lint issues
- Note any assumptions around copy, imagery, data, or navigation

## Output Expectations

When using this skill:

- Explain the design logic as well as the implementation changes
- Reference the files and components touched
- Call out assumptions about UX, copy, assets, and user flows
- Highlight premium details such as hierarchy, motion, and trust-building choices
- Mention follow-up work if the request needs assets, APIs, analytics, or content support
